# Workflow

This document describes the development workflow for this project.

1. **Create a new branch:** Create a new branch for each new feature or bug fix.
2. **Make changes:** Make your changes to the code.
3. **Run tests:** Run all the tests to make sure that your changes have not broken anything.
4. **Create a pull request:** Create a pull request to merge your changes into the main branch.
5. **Get a code review:** Get a code review from another developer.
6. **Merge the pull request:** Once the pull request has been approved, merge it into the main branch.
